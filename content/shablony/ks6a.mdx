КС-6а (журнал учёта выполненных работ) — первичный накопительный документ, в котором нарастающим итогом фиксируется объём выполненных работ по каждой позиции локальной сметы. Является основанием для составления [акта КС-2](/shablony/ks2). Форма утверждена **постановлением Госкомстата России №100 от 1999 года**.

## Что включено в шаблон

- Шапка с реквизитами объекта, заказчика и подрядчика
- Ссылка на локальную смету (номер, дата, сметная стоимость)
- Таблица: номер позиции, шифр по смете, наименование работ, единица измерения, количество по смете
- Колонки для каждого месяца строительства: выполнено за месяц, нарастающий итог
- Итоговые строки по разделам и всей смете
- Строки для подписей исполнителя и проверяющего

## Нормативная база

Форма КС-6а утверждена **постановлением Государственного комитета Российской Федерации по статистике №100** от 11 ноября 1999 года. Ведётся подрядчиком по каждой локальной смете отдельно и служит первичным основанием для КС-2.

<Callout type="info" title="КС-6а и КС-2: обязательная связка">
Данные КС-6а являются расчётной базой для КС-2. Строки в КС-2 должны соответствовать позициям КС-6а того же периода. Расхождения — сигнал для аудитора или инспектора о возможных манипуляциях с объёмами.
</Callout>

## Как вести КС-6а

<StepList>
  <Step number={1} title="Заполнение до начала работ">

  </Step>
  <Step number={2} title="Ежемесячное заполнение">

  </Step>
  <Step number={3} title="Нарастающий итог">
    Колонка «нарастающий итог» заполняется автоматически (или вручную) как сумма всех предыдущих месяцев включая текущий. Остаток = сметное количество минус нарастающий итог.
  </Step>
  <Step number={4} title="Привязка к КС-2">
    При формировании [КС-2](/shablony/ks2) берите данные за отчётный период из КС-6а. Суммы должны совпадать до копейки. Данные [ОЖР](/shablony/ozr) за тот же период должны подтверждать факт выполнения этих работ.
  </Step>
</StepList>

## Автоматизация учёта

При большом количестве позиций в смете ручное ведение КС-6а занимает значительное время. Платформа [Komplid](/pto) автоматически формирует КС-6а на основе данных из сметы и записей о выполнении работ, исключая ручной перенос данных и связанные ошибки.

## Частые вопросы

**Обязательно ли вести КС-6а, если есть КС-2?**

Формально КС-6а является первичным документом-основанием для КС-2, поэтому его ведение обязательно при расчётах по унифицированным формам. На практике некоторые заказчики принимают КС-2 без КС-6а — это нарушение порядка документооборота и риск при налоговой проверке.

**Сколько КС-6а нужно на объект?**

По одной на каждую локальную смету. Если по объекту несколько договоров — по одной на каждый договор и смету.

**Как исправить ошибку в КС-6а?**

Зачеркнуть неверную запись одной чертой (так чтобы её было видно), рядом написать правильное значение, поставить дату и подписи обеих сторон. Использование корректора не допускается.

**Можно ли вести КС-6а в Excel?**

Да. Excel-версия шаблона входит в комплект загрузки. Также существуют специализированные программы, в том числе [Komplid](/pto), где КС-6а формируется автоматически на основе внесённых данных о выполнении работ.

**Как КС-6а связан с общим журналом работ?**

[ОЖР](/shablony/ozr) фиксирует ежедневный ход работ в хронологическом порядке. КС-6а суммирует объёмы по видам работ и позициям сметы. При проверке инспектор сопоставляет записи в ОЖР с данными КС-6а — они должны быть согласованы.
